## Why Separate Script Tags?

**ES Module Behavior**:
- All `import` statements are hoisted
- Modules load in dependency graph order
- But execution is async and depth-first

**HTML Script Tag Behavior**:
- Script tags execute in document order
- Browser guarantees sequential execution
- Each script tag is a separate execution context

**Solution**: Load Motion in its own script tag BEFORE the module that needs it.
